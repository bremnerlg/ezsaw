# EZSAW 3.2.0A

## Goals with this Iteration

### Configurability Features
Now that a basic GUI has been established with a VIN entry option, the author is now going to attempt to implement a configurable transversal of the statistics in accordance with the requirements of the project. This is going to be done through JSON file(s) and the goal is for the following to be configurable

    - 1. The names of the table fields from which the data are to be drawn.
    - 2. The order in which the statistics are transversed as the 'next' and 'previous' buttons are selected.

Descriptors are also planned to be included in the  In order to improve the clarity and utility of the statistics found with each VIN lookup. This is to consist of the precise x and y values listed in parentheses, as well as the vehicle VIN, make, and model through a vehicle fetcher helper function.
