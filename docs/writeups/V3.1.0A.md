# EZSAW V3.1.0A

## Goals and Aims With This Iteration

### Abstract
Create a basic GUI that accepts a VIN with an EZ product entry in the database, and walk the user through a series of out-of-tolerance graphs representing the failed tests for the vehicle. The explanation component for troubleshooting will be ommitted for this iteration, as well as the configurable query generation. The goals, in short are:
    - 1. Accept a VIN input.
    - 2. Finds the out-of-tolerance entries for that VIN.
    - 3. Presents them as a graph in no particular order.

## Conclusion

### Graph Now Works (out of order)
The graphing utility has been tested and it now will display the first graph in a series of results. The ability to transverse the results is going to be implemented in the next iteration due to the desire for a configurable transversal of test results per Mr. Van Esch.

### PyQt Version Change
It was found that pygraphqt is completely incompatible with PyQt5, so the version was changed accordingly. This had no significant impact on the functionality or logic of the program.

### Configurability Potential
In order to make the analysis generic relative to the database being used for the graph drawing, the names of the tables and fields used for processing the graph data are going to be made configurable with version 3.2A. This is going to be done in the form of a .JSON converted to a python dictionary, from which the configurations are going to be drawn.
