# EZSAW V3.1.0A

## Goals and Aims With This Iteration

### Abstract
Create a basic GUI that accepts a VIN with an EZ product entry in the database, and walk the user through a series of out-of-tolerance graphs representing the failed tests for the vehicle. The explanation component for troubleshooting will be ommitted for this iteratoin, as well as the configurable query generation. The goals, in short are:
    - 1. Accept a VIN input.
    - 2. Finds the out-of-tolerance entries for that VIN.
    - 3. Presents them as a graph in no particular order.
